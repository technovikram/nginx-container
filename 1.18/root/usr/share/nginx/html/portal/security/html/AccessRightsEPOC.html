User Administration - Access Rights

Each Client-Central application has maintainable access rights. They are read, add, update, and delete privileges granted to users enabling them to perform certain functions within the applications. This page allows you to specify access rights for each application within the selected list on the Applications page for this user. Any changes to applications reflect on this page.

Note: Every institution has one designated primary security administrator. Maximum privileges applicable to the institution are defined on this user's security record. Other administrators and users can be given the same or lesser privileges. When adding or maintaining a user record, you are not able to give more access than you have. Please refer to the User Administration User Manual for further details.

Access Rights
This page allows you to specify application access rights.
Access Rights You can grant and revoke read, add, update, and delete access for the identified pages within each application.

Note: Only the set of applications that you selected on the Add Applications page display on this page.

Buttons
Select All This button automatically checks all allowable access rights for the applications listed.
Clear This button automatically unchecks all allowable access rights.
Reset This button resets access rights to the original values existing at the time the page was accessed.
Read Only This button automatically checks read (only) access rights for the identified pages within each application.
OK The OK button temporarily stores the information entered on this page and the Applications page, and returns you to the User Record page. You can then choose to save your changes by selecting Update. Before doing so, you have an opportunity to edit any information on any of the other pages.
Cancel The Cancel button discards your changes on this page, as well as any changes made on the Applications page, and returns you to the User Record page.
