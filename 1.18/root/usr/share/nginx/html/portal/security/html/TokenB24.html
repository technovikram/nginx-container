User Administration - Token

Each user must have a token assigned to their user ID to log onto www.fiserveft.com. This token provides a numeric password that ensures secure access. If the user's token is lost, stolen, damaged, or defective, you can either assign an existing token that you possess to the user, or replace the token with a new one. Based on your selection, the information you are asked to provide varies.

Token Information
If the user's token is lost, stolen, damaged, or defective, this section allows you to request a new token for the user or reassign an existing token. Screen content varies based on your selection.
Token Serial Number Serial number of the user's token.
Request Token Initiates request to replace the user's token or assign an existing token to the user. Valid values are:
  • None
  • Replace Token
  • Assign Existing Token
Note: To assign an existing token, you must delete the user security record of the original owner of the token first. A token can only be associated with one user record and cannot be shared.
Replacement Reason Replacement Reason is a required field that only displays when Replace Token is selected in the Request Token field. Valid values are:
  • Lost/Stolen
  • Damaged
  • Defective

Shipping Address
This section displays only if you choose the Replace Token option. The new token is shipped to the address you provide. Providing all requested information helps to ensure that the token gets shipped as early as possible.
Option: Administrator / User / Default / Other For better efficiency you can auto-fill the shipping address by selecting any one of the options as detailed below:

Administrator - Selecting this option auto-fills the address fields with the address information from your user security record.

User - Selecting this option auto-fills the address fields with the address information from the details page of this user.

Default - This application provides a feature for you to store a default shipping address for the institution. Selecting this option auto-fills the address fields with the address specified as default for the institution.

Other - To provide a different shipping address, selecting this option clears the fields allowing you to enter the relevant information.

Institution Name of the institution to which the token is being shipped.
Deliver To Name of the person designated to receive the token.
Address Shipping address.

Note: This must be a physical address, not a P.O. box.

Country Destination country of the shipment.
City Destination city of the shipment.
State Destination state of the shipment.
ZIP Destination ZIP code of the shipment.
Phone Phone number of the person designated to receive the token.
Ext Extension of the person designated to receive the token.
Fax Fax number of the person designated to receive the token.
Email Email address of the person designated to receive the token.

Buttons
OK The OK button temporarily stores the information entered and takes you to the User Record page. You can then choose to save your changes by selecting Update or you can make additional changes on any of the other tabs available. All changes are held until you select Update.
Cancel The Cancel button discards your changes and returns you to the User Record page.
