User Administration - Add Tokens

Each user must have a token assigned to their user ID to log onto www.fiserveft.com. This token provides a numeric password that ensures secure access. As part of the add user process, you can either assign an existing token to the new user, or issue a new token. Based on your selection, the information you are asked to provide varies.

Token Information
Allows you to specify whether to issue a new token for the new user or reassign an existing token. Screen contents vary based on your selection.
Request New Token Provides access to the fields necessary to request a new token for the user being added.
Assign Existing Token Provides access to the fields necessary to request that an existing token be assigned to the user being added.

Note: To assign an existing token, you must delete the user security record of the original owner of the token first. A token can only be associated with one user record and cannot be shared.

Shipping Address
Displays when you select the Request New Token radio button. The new token is shipped to the address you provide. Providing all requested information helps to ensure that the token gets shipped as early as possible.
Option: Administrator / User / Default / Other / Copied User For better efficiency you can auto-fill the shipping address by selecting any one of the options as detailed below:

Administrator - Selecting this option auto-fills the address fields with the address information from your user security record.

User - Selecting this option auto-fills the address fields with the address information from the details page of this new user.

Default - This application provides a feature for you to store a default shipping address for the institution. Selecting this option auto-fills the address fields with the address specified as default for the institution.

Other - To provide a different shipping address, selecting this option clears the fields allowing you to enter the relevant information.

Note: This option does not display when pages are accessed via the Copy User button on User Administration - Choose User.

Copied User - Selecting this option auto-fills the address fields with the address information as existing on the details page of the copied user's security record.

Note: Copied User displays only when pages are accessed via the Copy User button on User Administration - Choose User.

Buttons
Back The Back button takes you to the previous page.

Clicking Back from the Add Tokens page takes you to the Add Availability page.

Continue The Continue button temporarily stores the token information specified on the new user's security record and takes you to the next page.

Clicking Continue from the Add Tokens page takes you to the Selected User page.

Cancel The Cancel button discontinues the add user process and returns you to the Choose User page.

Clicking Cancel from the Add Tokens page displays the following confirmation message:
Click OK to exit Add User.
