User Administration—User Details

You can view and change basic information relating to the user—mainly the user's name and address—on this page. Some fields on the page cannot be changed and are identified below.

Note: Every institution has one designated primary security administrator (PSA). Maximum privileges applicable to the institution are defined on the PSA's security record. Other administrators and other users can be given the same or lesser privileges. When adding or maintaining a user record, you are not able to give more access than you have. Please refer to the Security manual for further details.

User Information
Lists basic information for the selected user.
System Field displays when the Administrator has access to both EPOC and Full Service Credit Systems. It allows the Administrator to set up a user with one or both systems. Field is defaulted to EPOC when it does not display on this page. System controls access to Documents, Tutorials and Forms in Client Central as well as system specific fields within User Administration.
First Name First name of the user.
Last Name Last name of the user.
Institution ID 6-digit identifier assigned by Card Services for the selected institution. This field defaults based on your user ID and cannot be changed.

Note: Administrators can only manage security records for the institution identified in this field on their user record.

ID Information
User ID The user ID assigned to this user record.  Used for logging in to Client-Central.  Once the user record has been added, the user ID cannot be changed.
LNET Identifies the logical network to which the user is granted access. This field cannot be changed. Valid values are PRO3 or SCS4.
Alias Unique user ID used by back-end systems to validate a user record. It is an auto-generated value that cannot be changed and generally matches the user ID.
Group Number In addition to the institution ID, this is a unique number used by back-end systems to associate a user record with an institution. It is an auto-generated value that cannot be changed and generally matches the institution ID.
User Number Associated the user ID, this is a unique number used by back-end systems for user identification and authentication. It is an auto-generated value that cannot be changed.
VisionPLUS ID Field visible only when Full Service Credit System is checked. Used to contain the VisionPLUS user ID for Single Sign On into Credit Console, and other Full Service Credit applications as applicable.
Regions MultiSelect List box (highlight all applicable regions). Field visible only when Full Service Credit System is checked. Used to control the VisionPLUS Regions available to the user.
