User Administration—Reports

This page allows you to view information pertaining to service requests (adds, changes, and token requests) done by, or for, your institution in the past 90 days.

The Search section allows you to enter information into one or more of the available fields and run various searches to display specific maintenance done in the CWSi User Administration application. When choosing the print link, you will be given the search criteria as well as the results.

Use the buttons and tabs provided in this application for navigation. The Enter key is not a valid option and may give you unexpected results.

Search
This section allows you to search for user security record maintenance done within the last 90 days. You can leave the search criteria blank, use a single field, or any combination of the available fields for your search.
Institution ID Displays the financial institution ID associated with your token ID.
User ID

Allows you to search for a user security record by user ID.

Note: Partial data cannot be entered but the field can be left blank.

User Request Type Allows you to search for specific service request types.

The drop-down list contains the following options:

  • None
  • Add user
  • Delete user
  • Update user
Token Request type Allows you to search for a specific kind of token request. The options available in the drop-down list depend on the User Request Type selected.

When User Request Type is set to None, the list contains:

  • None
  • Issue New Hard Token
  • Unassign Hard Token
  • Replace Hard Token
  • Assign Existing Hard Token
  • Issue New Soft Token
  • Remove Soft Token
  • Activate Soft Token
  • Expire Soft Token

When User Request Type is set to Add User, the list contains:

  • None
  • Issue New Hard Token
  • Assign Existing Hard Token
  • Issue New Soft Token

When User Request Type is set to Delete User, the list contains:

  • None
  • Unassign Hard Token
  • Remove Soft Token

When User Request Type is set to Update User, the list contains:

  • Issue New Hard Token
  • Unassign Hard Token
  • Replace Hard Token
  • Assign Existing Hard Token
  • Issue New Soft Token
  • Remove Soft Token
  • Activate Soft Token
  • Expire Soft Token
Requestor User ID

Allows you to search for service requests done by a specific administrator.

Note: Partial data cannot be entered but the field can be left blank.

Status Allows you to search for service requests with a specific status.

Options:

  • None
  • Completed
  • In progress
User First Name

Allows you to search for maintenance done to a user security record based on the user's first name.

Note: Partial data cannot be entered but the field can be left blank.

User Last Name

Allows you to search for maintenance done to a user security record based on the user's last name.

Note: Partial data cannot be entered but the field can be left blank.

Request Date From

Allows you to specify a date range for your search. This field defaults to 90 days prior to the current date but can be changed.

Format: MM/DD/YYYY

Request Date To

Allows you to specify a date range for your search. This field defaults to the current date but can be changed.

Format: MM/DD/YYYY

Request Report
This section displays a read-only results list in table format based on the search criteria entered. Each row represents a service request.
Request ID A unique identifier for each add, change, or token request.
User ID User ID being added or changed.
User Name Name associated with the user ID.
Request Type Type of action or request, for example, Add User or Issue New Token.
Applications Displays the Applications and Access Rights access that were added or changed.
Request Date (ET) Date and time the service request (add user, change user, or token request) was done.
Requestor ID User ID of the person adding the record or making the change.
Status Current status of the request. Will show In progress until it is completed.
Deliver To Token delivery information, applicable only when a new token is requested.
