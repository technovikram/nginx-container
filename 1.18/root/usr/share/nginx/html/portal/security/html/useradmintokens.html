User Administration - Tokens can be accessed as follows:
  1. Click the Continue button on User Administration - Availability.

    2. - or -
  2. Click Tokens on the User Record tab within User Administration - Navigation.

User Administration - Tokens allows you to specify token issuance information.

User Administration - Tokens contains the following information:

Key Information (contains view-only fields)

  • First Name – First name of the user.
  • Last Name – Last name of the user.
  • Logo – Four-character alphanumeric client identifier.
  • User ID – Client-Central user ID of the user.
Note: The user ID displays only when an existing user record is being accessed.

Token Information (contains modify fields)

The screen contents vary based on the selection below:

  • Issue Token – Initiates request for a new token to be issued to the user being added to Client-Central.
  • Reassign Token – Initiates request to reassign an existing token to the user being added to Client-Central.
    Note: To reassign a token, you must delete the user security record of the original owner of the token.
  • Replace Token – Initiates request to replace the token for an existing user.
  • Token Serial Number – Serial number of the token allocated to the user.
  • Replacement Reason – Reason for the token replacement request.
Note: Replace Token, Token Serial Number, and Replacement Reason display only when an existing user record is being accessed.

Original Owner Information (contains modify fields)

  • Token Serial Number – Serial number of the token allocated to the original user.
  • First Name – First name of the original owner of the token.
  • Last Name – Last name of the original owner of the token.
  • User ID – Client-Central user ID of the original owner of the token.

Note: The Original Owner Information section displays only if Reassign Token is selected.

Shipping Address (contains view-only and modify fields)

  • Administrator – Selecting this option prefills the address fields with the address of the administrator (person adding or editing this record).
  • User – Selecting this option prefills the address fields with the user address specified on User Administration - User Record.
  • Default – Selecting this option prefills the address fields with the address specified in the Shipping Address section on User Administration – Defaults.
  • Other – Selecting this option allows the user to specify a different shipping address.
    Note: The Other option does not display when pages are accessed via the Copy User button on User Administration - Choose User.
  • Copied User – Selecting this option prefills the address fields with the user address specified in the copied user's security record.
    Note: The Copied User option displays only when pages are accessed via the Copy User button on User Administration - Choose User.
  • Institution – Name of the institution to which the token is being shipped.
  • Deliver To – Name of the person designated to receive the token.
  • Address – Shipping address.
  • Country – Destination country of the shipment.
  • City – Destination city of the shipment.
  • State – Destination state of the shipment.
  • ZIP – Destination ZIP code of the shipment.
  • Phone – Phone number of the person designated to receive the token.
  • Ext – Extension of the person designated to receive the token.
  • Fax – Fax number of the person designated to receive the token.
  • Email – E-mail address of the person designated to receive the token.

Comments (contains modify fields)

  • Comments – Comments that may be useful when processing the token request.

Buttons, links, and icons contained in User Administration - Tokens:

Buttons
Back – Takes you back to User Administration - Availability.

Continue – Takes you to User Administration - Navigation.

Cancel – Takes you to User Administration - Choose User or User Administration - Navigation without saving changes.

OK – Allows you to temporarily save the modifications made to the page. Click the Save or Update button on User Administration - Navigation to permanently save your changes.

Note: The Back and Continue buttons display when pages are accessed from the Add button on User Administration - Choose User. Otherwise, the OK button displays.
