User Administration - Special Roles

In addition to the normal read, add, delete, and update permissions, you set up special roles within an application. A special role must be assigned in order for the user to be able to perform that specific task. If a user does not have a special role, even if they have full access to the rest of the permissions for the application, they are not able to perform that specific task.

The special roles that display on this page depend on the selected applications given to the user who is being managed, as well as the special roles that you have access to. This page allows you to manage user access to the available special roles. You can select and move one or more of these special roles between the available and the selected lists. The user is granted access to the roles in the Selected Roles list.

Note: Every institution has one designated primary security administrator (PSA). Maximum privileges applicable to the institution are defined on this user's security record. The PSA can give other administrators and other users the same or lesser privileges. When adding or maintaining a user record, you are not able to give more access than you have. Please refer to the EFT User Manual, User Administration for details.
