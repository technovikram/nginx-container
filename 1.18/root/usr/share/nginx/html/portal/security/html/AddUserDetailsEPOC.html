User Administration—Add User Details

The first step in the add user process is to enter basic information relating to the new user—mainly the user's name and address. Some fields on the page default and cannot be changed.

Note: Every institution has one designated primary security administrator. Maximum privileges applicable to the institution are defined on this user's security record. Other administrators and other users can be given the same or lesser privileges. When adding or maintaining a user record, you are not able to give more access than you have. Please refer to the Security manual for further details.

Also, if you have chosen the Copy User option, most of the information on this page and the subsequent pages is copied from the selected record. This information can be modified as needed for the new user being added.

User Information
Allows entry of basic information relating to the new user.
System Field displays when the Administrator has access to both EPOC and Full Service Credit Systems. It allows the Administrator to set up a user with one or both systems. Field is defaulted to EPOC when it does not display on this page. System controls access to Documents, Tutorials and Forms in Client Central as well as system specific fields within User Administration.
First Name First name of the user whose record is being created.
Last Name Last name of the user whose record is being created.
Institution ID 4-character alphanumeric identifier assigned by Card Services for the selected institution. This field defaults based on your user ID and cannot be edited.

Note: Administrators can only add and manage security records for the institution identified in this field on their user record.

ID Information
User ID

Generally auto-generated and does not normally display. If it does display, your institution has been set up to allow custom user IDs. If this field displays and is left blank, the user ID is auto-generated. If the field displays and you enter the user ID, be sure to use the guidelines established by your institution for assigning user IDs.

Note: If this field does display, the Administrator has been set up to allow custom user IDs. When this is the case, you have the following options:

  • Enter a user ID based on the guidelines established by your institution for assigning user IDs.
  • Leave the field blank and it will be generated automatically.
Group Number Client-Central systems support multiple layers of security. In addition to the institution ID, this is a unique number used by back-end systems to associate a user's security record with an institution. This is an auto-generated field that cannot be edited.
User Number Associated with the user ID, this is a unique number used by back-end systems for user identification and authentication. This is an auto-generated field that cannot be edited.
VisionPLUS ID Field visible only when Full Service Credit System is checked. Used to contain the VisionPLUS user ID for Single Sign On into Credit Console, and other Full Service Credit applications as applicable.
Regions MultiSelect List box (highlight all applicable regions). Field visible only when Full Service Credit System is checked. Used to control the VisionPLUS Regions available to the user.
