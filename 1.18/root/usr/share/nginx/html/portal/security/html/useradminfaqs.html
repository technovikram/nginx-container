Access User Administration

1 Q:   How do I access User Administration?
  A:  
  1. Log on to Client-Central.

  2. Click User Administration on the gray navigation column.
2 Q:   How can I access user security records in User Administration?
  A:  
  1. On the User Administration - Choose User page, enter relevant search criteria and click the Search button.
    The records returned by the search are displayed in the Results table.
    Note: You may use any combination of fields to enter search criteria.
  2. Select a record to view details, or print, update, or delete the user security record.

Administrators

3 Q:   What is the role of a primary administrator?
  A:   A primary administrator is set up for your institution during the implementation of Client-Central. There is only one primary administrator in an institution. The primary administrator can set up other administrators and users by granting them read, add, update, and delete access to the User Administration application.
4 Q:   What is the difference between the primary administrator and other administrators?
  A:   Only primary administrators can add, update, and delete an administrator's user security record. They can also edit the first name, last name, and the address information on their own record. Other administrators are allowed to add, update, and delete user security records for users belonging to their institution.
5 Q:   Can there be mutiple administrators in an institution?
  A:   Yes. User Administration supports multiple administrators.

Add or Copy Users

6 Q:   How do I add user security records in User Administration?
  A:  
  1. On the User Administration - Choose User page, click the Add button.
  2. Fill in the required information in the pages that are displayed as you continue through the Add process.
  3. Select the option Charges will apply as set forth in your Client-Central pricing documentation on the User Administration - Navigation page.
  4. Click the Save button.
    The user security record is created and the User Administration - Confirmation page displays.
    Note: You may print the confirmation page by clicking the Print Page link.
  5. Click the Choose User tab.
    The User Administration - Choose User page displays.
7 Q:   When do I select the Set Same as Yourself option?
  A:   Select the Set Same as Yourself option if the address of the user being added is the same as the address of the admnistrator adding the record. You may edit this information if necessary.
8 Q:   How can I grant a new user the same rights and settings as an existing user?
  A:  
  1. Follow Step 1 contained in FAQ 2.
  2. Select the relevant existing user's security record.
  3. Click the Copy User button on the User Administration - Choose User page.
    A new user security record is displayed with the same rights and settings as the selected user.
  4. Fill in the required information in the pages that are displayed as you continue through the Add process.
  5. Click the Save button.
    The user security record is created and the User Administration - Confirmation page displays.
    Note: You may print the confirmation page by clicking the Print Page link.
9 Q:   What are the applicable charges for creating, editing, and deleting users in the User Administration application?
  A:   All applicable charges are defined in the authorized service request submitted during the implementation of Client-Central for your institution. The service request defines the pricing structure for your institution. By accepting the statement Charges will apply as set forth in your Client-Central pricing documentation, you confirm that all relevant Client-Central pricing as defined for your institution is acceptable.

Manage Users

10 Q:   How do I manage user security records in the User Administration application?
  A:  
  1. On the User Administration - Choose User page, enter relevant search criteria.
  2. Click the Search button.
    The records returned by the search are displayed in the Results table.
  3. Select the relevant record.
  4. Click the Manage User button.
    The User Administration - Navigation page displays where you may update or delete the selected user security record.

Tokens

15 Q:   How do I request a token for a new user?
  A:  
  1. Follow Steps 1 and 2 contained in FAQ 6 to add a security record for the new user.
  2. Select Issue token on the User Administration - Tokens page.
  3. Provide the shipping address.
  4. Save the changes.
    The process to issue a token is initiated.
16 Q:   How do I reassign an existing token to a new user user being created?
  A:  
  1. Delete the user security record of the original user.
  2. Follow Steps 1 and 2 contained in FAQ 6 to add a security record for the new user.
  3. Select Reassign token on the User Administration - Tokens page.
  4. Provide the original owner's information.
  5. Save the changes.
    The process to reassign an existing token is initiated.
17 Q:   How do I request a replacement token for a lost, defective, or damaged token?
  A:  
  1. Follow Steps 1 and 2 contained in FAQ 11 to update a user security record.
  2. Select the relevant user's record.
  3. Select Replace Token on the User Administration - Tokens page.
  4. Provide the shipping address.
  5. Save the changes.
    The process to replace a lost, defective, or damaged token is initiated.
18 Q:   What are the four options available under Shipping Address?
  A:   The four available options that specify the shipping address are as follows:
  1. Administrator – The token is shipped to the address of the administrator creating the request.
  2. User – The token is shipped to the user's address as defined on the User Administartion - User Details page.
  3. Default – The token is shipped to the shipping address as defined on the User Administration - Defaults page.
  4. Other – The token is shipped to the address you specify.

General

19 Q:   Can I check the status of the administrative tasks performed?
  A:   Yes. You can view the status by selecting Request Status under the Reports tab. This report contains information on all open tasks, as well as on tasks completed in the past 30 calendar days.
20 Q:   When does a user's security access rights become effective?
  A:   The anticipated time frame for a users security access rights to take effect is approximately seven working days. Transit duration for the token depends on the mode of shipping.

Note: Updates to CWSi access rights for existing users take effect immediately. Deletion of a user security record takes effect immediately and the deleted user can no longer access Client-Central.
