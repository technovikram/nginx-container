User Administration—Add Tokens

Each user must have at least one hard or soft token assigned to their user ID to log on to the secure client website. This token provides a numeric password that ensures secure access. As part of the add user process, you can either assign an existing token to the new user or issue a new hard or soft token. Based on your selection, the information you are asked to provide varies.

Note: Your institution must be set up for soft tokens before the Soft Token Information section is displayed.

Token Selection
Section displays to identify the type of token being requested for the new user. If your institution is set up to allow Soft Tokens you will be given a drop down list containing all available options. If not, Token Type will simply be displayed as Hard Token.
Token Type

Required. Displays as Hard Token unless your institution is set up to allow Soft Tokens. In that case you will be given a drop down listing containing the following options.

Values:

  • Select One (default)
  • Hard Token—Displays the Hard Token Information section, allowing you to order a Hard Token for the new user.
  • Soft Token—Displays the Soft Token Information section, allowing you to create a Soft Token request for the new user.
  • Hard and Soft Token—Display both the Hard Token Information and Soft Token Information sections. Select this option if you want to order both a Hard and Soft Token for the new user.

Soft Token Information
Displays when you select the Soft Token or Hard and Soft Token option from the Token Type drop-down list. Creates a soft token request for the new user which generates two Security Token Assignment emails—one with the User ID and link to the activation portal and another with the activation code. Both are sent to the email address in this section.
Email

Email address provided in this field is used for the two Security Token Assignment emails generated for the soft token request. Defaults to the email address from the User Administration—User Details page.

Note: Any changes made to this field also update the email address on the User Administration—User Details page.

Hard Token Information
Displays when you select the Hard Token or Hard and Soft Token option from the Token Type drop-down list. Allows you to specify if the new user will use an existing token or be issued a new token. Screen contents vary based on your selection.
Issue New Token Radio button. When selected, provides a page containing all of the necessary fields for requesting a new hard token.
  • Shipping Address section
  • Comments
Assign Existing Token

Radio button. When selected, provides a page containing all of the necessary fields for reassigning an existing hard token.

  • Original User Information section
  • Comments

Note: To assign an existing token, you must delete the user security record of the original owner of the token first. A token is associated with only one user's record and cannot be shared.

Shipping Address
Displays when you select the Issue New Token radio button. The new token is shipped to the address you provide. Providing all information helps to ensure that the token gets shipped as early as possible.
Administrator/User/Default/Other/Copied User

Radio buttons. Auto-fills the shipping address by selecting one of the options below:

  • Administrator—Default. Hard token will be shipped to the address associated with your user security record.
  • User—Hard token will be shipped to the address from the User Administration—User Details page for this user.
  • Default—Hard token will be shipped to the institution's default address.
  • Other—All address fields in this section are blank and editable. Hard token will be shipped to the address that is entered.
  • Copied User—Hard token will be shipped to the address of the user that had been copied. This option only displays when the add user pages are accessed via the Copy User button on User Administration—Choose User.

Original Owner Information

Hard Token Serial Number

Serial number of the hard token to be reassigned.

Note: A hard token can only be associated with one user record at a time. You would typically delete the original user record before reassigning the token but if the serial number is still assigned to another user record, you will be given the following message on this page: Please check the token serial number entered. It is currently assigned to user x. To remove this token serial number from user x, click Remove.
