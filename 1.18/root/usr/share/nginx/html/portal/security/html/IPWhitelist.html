User Administration—IP Whitelisting

Overview
This Self-Service IP Whitelisting tab allows your Security Administrators to set up the IP addresses where your users are able to log in to Client Central. Read access to the User Administration application allows a user to view the page. Security Administrators must also have Add, Update, and Delete access to maintain the data on the page.

Authorized Incoming IP Address
Count Counts the number of Authorized Incoming IP addresses in the list.
IP Address Displays the Individual IP address or IP address range.
Description Displays the description associated with the IP address or IP address range.
Added By Displays the User ID of the person who added the IP address data.
Date/Time Displays the Date/Time the IP address data was added for the institution.
Action A Delete link is available for users who have Delete access to User Administration. It allows the IP address data to be removed.
Note: To edit IP address data, it must be deleted and re-added.

IP User Restrictions
Default all new users to be restricted by the above authorized IPs checkbox When checked, all new users will be given the IP Address Restriction special role. This will require them to log in from one of the authorized IP addresses.
When unchecked, the IP Address Restriction special role will be listed in the Available Special Roles column and can be given to a user manually during the add process.
Users without restrictions Displays the users that have not been given the IP Address Restriction special role. They will not be restricted by the authorized IP addresses.
Note: It is recommended that at least one Security Administrator at the institution is set up in this manner.
Restricted Users Displays the users that have the IP Address Restriction special role and can only log in using one of the authorized IP addresses.
Arrows Allows the Security Administrator to move user records between the two boxes, adding or removing the IP Address Restriction special role from the user record.
Note: This can also be done at the User record level using the Special Roles page in Add or Manage User.
