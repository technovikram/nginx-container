User Administration—Token

Each user must have at least one hard or soft token assigned to their user ID to log onto the secure client website. This token provides a numeric password that ensures secure access. If the user needs additional soft tokens (one user record can have up to 25) or if the user's hard token is lost, stolen, damaged, or defective, use this page to manage all token requests. Based on the radio button selected, the page will change as needed to provide the information required to complete your request.

Note: Your institution must be set up for soft tokens before the Soft Token Information section is displayed.

Soft Token Information
This section provides the information required to view and manage a user's soft tokens. It only displays when the institution has been set up to use soft tokens.
Remove

Check box which can be used to remove one or more soft tokens.

Note: This would typically be used when a user forgets a soft token password. In that situation, the soft token record is removed and a new soft token is requested.
Serial Numbers

Listing of all soft tokens assigned to the user. Displays as Pending until the soft token has been activated. Once activated, the serial number displays.

Note: User record can have up to 25 soft tokens.

Generate Soft Token

Creates a new soft token request for the user. Two Security Token Assignment emails are generated—one with the User ID and link to the activation portal and another with the activation code. Both are sent to the email address on this page.

Note: You should not assign new soft token until the previous soft token has been activated. If there is a pending soft token request, the following message displays with OK and Cancel options when this box is checked: User has a pending soft token request. If it has not been activated, requesting another soft token will generate an email with a new activation code that replaces the one provided previously.
Email

Displays when the Generate Soft Tokens box is checked and is used for the soft token emails. Defaults to the email address from the User Administration—User Details page.

Note: Any changes made to this field also update the email address on the User Administration—User Details page.

Hard Token Information
This section provides the information required to view and manage a user's hard tokens.
Token Serial Number Serial number of the user's hard token. Displays as Pending if a new hard token has been requested but not yet been assigned. This field will not display if the user record does not have a hard token.
None Default. Radio button that indicates there will be no requests for hard tokens at this time.
Request Replacement Hard Token

Radio button. When selected, provides a page containing all of the necessary fields for requesting a new hard token.

  • Replacement Reason
  • Shipping Address section
  • Comments
Assign Existing Hard Token

Radio button. When selected, provides a page containing all of the necessary fields for reassigning an existing hard token.

  • Original User Information section
  • Comments

Replacement Reason
This field displays when you choose the Request Replacement Hard Token radio button.

Replacement Reason

Replacement Reason is a required field that allows you to identify what happens to the original hard token.

Values:

  • Lost/Stolen
  • Damaged
  • Defective

Shipping Address
This section displays when you choose the Request Replacement Hard Token radio button. The new hard token is shipped to the address you select. Providing all requested information helps to ensure that the token gets shipped as early as possible.
Administrator/User/Default/Other

Radio buttons. Auto-fills the shipping address fields as indicated below. The Other radio button is the only option which provides editable fields.

  • Administrator—Default. Hard token will be shipped to the address associated with your user security record.
  • User—Hard token will be shipped to the address from the User Administration—User Details page for this user.
  • Default—Hard token will be shipped to the institution's default address.
  • Other—All address fields in this section are blank and editable. Hard token will be shipped to the address that is entered.
Institution Name of the institution to which the token is being shipped.
Deliver To Name of the person designated to receive the token.
Address Shipping address.

Note: This must be a physical address, not a P.O. box.

Country Destination country of the shipment.
City Destination city of the shipment.
State Destination state of the shipment.
ZIP Destination ZIP code of the shipment.
Phone Phone number of the person designated to receive the token.
Ext Extension of the person designated to receive the token.
Fax Fax number of the person designated to receive the token.
Email Email address of the person designated to receive the token.

Original User Information
This section displays when you choose the Assign Existing Hard Token radio button. Information requested relates to the original user of the token.
Hard Token Serial Number Serial number of the hard token to be reassigned.

Note: A hard token can only be associated with one user record at a time. You would typically delete the original user record before reassigning the token but if the serial number is still assigned to another user record, you will be given the following message on this page: Please check the token serial number entered. It is currently assigned to user x. To remove this token serial number from user x, click Remove.
First Name First name of the original owner of the token.
Last Name Last name of the original owner of the token.
User ID User ID of the original owner of the token.
